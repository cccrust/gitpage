# v0.8 — Build/Deploy Log Viewer（GitHub Actions 風格）

讓使用者可以查看 app 每次部署的完整日誌（build + start），類似 GitHub Actions 的 run 頁面。每次手動或自動部署都會建立一條記錄，包含狀態、時間戳與完整輸出。

---

## 架構概覽

```
觸發部署（git push / 手動 Redeploy / 啟用 App）
        ↓
   create_deploy_log(repo_id)  → status = 'running', started_at = now
        ↓
   deploy_app()  → 逐步累積 log 文字
   ├─ checkout source
   ├─ 偵測專案類型
   ├─ run_build (stdout+stderr)
   └─ start_app (port)
        ↓
   update_deploy_log(id, status, log_output)  → status = 'success' | 'failed'
        ↓
   DB deploy_logs 表中有完整紀錄
        ↓
   API GET /api/apps/:repo_id/deploys     → 列表（最新 20 筆）
   API GET /api/apps/:repo_id/deploys/:id → 單筆明細（含完整 log）
```

## 1. Database

### deploy_logs 表

```sql
CREATE TABLE IF NOT EXISTS deploy_logs (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    repo_id     INTEGER NOT NULL REFERENCES repositories(id),
    status      TEXT NOT NULL DEFAULT 'running',
    started_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    finished_at DATETIME,
    log_output  TEXT DEFAULT ''
);
```

| 欄位 | 說明 |
|------|------|
| `id` | 自動遞增主鍵 |
| `repo_id` | 外鍵，指向 repositories |
| `status` | `'running'` / `'success'` / `'failed'` |
| `started_at` | 部署開始時間 |
| `finished_at` | 部署完成時間（NULL 表示進行中） |
| `log_output` | 完整 build + start 日誌文字 |

### CRUD 函式

| 函式 | 說明 |
|------|------|
| `create_deploy_log(repo_id)` | 新建一筆 status='running' 記錄 |
| `update_deploy_log(id, status, log_output)` | 設定 status、log_output、finished_at |
| `append_deploy_log(id, log_output)` | 追加日誌到現有 log_output（保留供後續串流使用） |
| `get_deploy_logs(repo_id)` | 回傳該 repo 最新 20 筆，按 started_at DESC |
| `get_deploy_log(id)` | 回傳單筆記錄 |

## 2. 部署日誌流程

### deploy_app 回傳型別

```
pub async fn deploy_app(...) -> Result<(u16, String), AppError>
                                    (port,  log)
```

- `log` 是一個累積的 `String`，每個步驟都會 append：
  - `"[checkout] Checked out source to {workspace_dir}\n"`
  - `"[detect] Detected Node.js project\n"`
  - `"[build] npm install 的 stdout+stderr\n"`
  - `"[start] App started on port {port}\n"`
- `run_build()` 回傳 `String`（stdout+stderr 合併），也 append 進 `log`

### do_deploy 輔助函式

```
async fn do_deploy(state, repo_id, user_id) -> Result<...>
  1. create_deploy_log(repo_id)  → 得到 deploy_log.id
  2. deploy_app(...)
     Ok((port, log))  → update_deploy_log(id, "success", log)
     Err(e)           → update_deploy_log(id, "failed",  log + error)
  3. 回傳 success/false + port + url + deploy_log_id
```

被 `update_apps_config`（啟用時自動部署）與 `deploy_apps_handler`（手動 Redeploy）呼叫。

### auto_deploy_app

git push 後的背景任務，流程相同：

```
auto_deploy_app(state, username, repo_name)
  → 查詢 user / repo / apps_config
  → create_deploy_log(repo.id)
  → deploy_app(...)
  → update_deploy_log(id, "success"|"failed", log)
```

## 3. API

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | /api/apps/:repo_id/deploys | 列出該 repo 最新 20 筆部署日誌 |
| GET | /api/apps/:repo_id/deploys/:deploy_id | 取得單筆日誌詳細內容 |

### GET /api/apps/:repo_id/deploys response

```json
{
  "deploy_logs": [
    {
      "id": 1,
      "repo_id": 3,
      "status": "success",
      "started_at": "2026-06-16T12:00:00Z",
      "finished_at": "2026-06-16T12:00:15Z",
      "log_output": ""
    }
  ]
}
```

列表只回傳 metadata，不包含 `log_output`（內容可能很大）。

### GET /api/apps/:repo_id/deploys/:deploy_id response

```json
{
  "deploy_log": {
    "id": 1,
    "repo_id": 3,
    "status": "failed",
    "started_at": "2026-06-16T12:00:00Z",
    "finished_at": "2026-06-16T12:00:10Z",
    "log_output": "[checkout] Checked out source to data/apps/alice/hello-node/source/\n[detect] Detected Node.js project\n[build] npm install\nnpm ERR! ...\n"
  }
}
```

## 4. 前端 DeployLogsPage（`/repo/:id/deploys`）

```
~ / hello-node / App / Deploy Logs

  狀態列表：
  ┌──────────────────────────────────────────┐
  │ ✓ success    2026-06-16T12:00:00Z    →  │
  │ ✗ failed     2026-06-16T11:50:00Z    →  │
  │ ⏳ running   2026-06-16T11:40:00Z    →  │
  └──────────────────────────────────────────┘
```

- 每筆顯示：狀態圖示 + 狀態文字 + 時間
- 點擊進入明細頁
- 空白狀態：「No deployments yet.」

### DeployLogDetailPage（`/repo/:id/deploys/:deployId`）

```
~ / hello-node / App / Deploy Logs / #1

  ✓ Deploy #1
  Status: success  |  Started: 2026-06-16T12:00:00Z  |  Finished: 2026-06-16T12:00:15Z

  ┌─────────────────────────────────────┐
  │ [checkout] Checked out source to... │
  │ [detect] Detected Node.js project   │
  │ [build] npm install                 │
  │ npm notice created lockfile...      │
  │ [start] App started on port 4052    │
  └─────────────────────────────────────┘
```

- 深色背景 `<pre>` 區塊顯示完整 log_output
- status 用顏色區分（綠色 success、紅色 failed、黃色 running）
- 從 AppSettingsPage 的「Deploy Logs」按鈕導航進來

## 6. 注意事項

- 目前 log 是部署完成後一次性寫入，非即時串流。若部署時間長（如 npm install 很久），使用者需要等完成才能看到日誌。
- log_output 存在 SQLite TEXT 欄位，無大小限制但大量日誌可能影響查詢效能。v0.9 可考慮搬至檔案系統或串流。
- `append_deploy_log` 函式已實作但尚未使用，保留供未來即時串流使用。
- 伺服器重啟後，之前的部署記錄仍保留在 DB 中，不會遺失。
- 列表最多回傳 20 筆，避免 frontend 載入過多資料。需要完整歷史可實作分頁。
