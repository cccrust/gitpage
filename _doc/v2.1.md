# v2.1 — 後台設定管理

Access Token、協作者、Secrets、Branch Protection 統一設定管理。

## 動機

- 先前的設定（repo 基本資料、Pages、App）已可透過網頁操作
- 但 Access Token、協作者、CI/CD Secrets、分支保護仍需手動處理
- v2.1 補齊這四項，讓 gitpage 不需要直接操作 config.toml 即可管理大部分設定

## 新增功能

### Access Token

Personal Access Token，用於 API 認證（類似 GitHub Token）。

| 功能 | API | 前端頁面 | 狀態 |
|------|-----|----------|------|
| 列出 Token | `GET /api/user/tokens` | `/settings/tokens` | ✅ |
| 建立 Token | `POST /api/user/tokens` | `/settings/tokens` | ✅ |
| 刪除 Token | `DELETE /api/user/tokens/:token_id` | `/settings/tokens` | ✅ |

**Token 格式**：`gpt_<40字元隨機字母數字>`，明碼只顯示一次。SHA-256 hash 存 DB。

### Collaborators（協作者）

讓非 owner 的使用者擁有 repo 寫入權限。

| 功能 | API | 前端頁面 | 狀態 |
|------|-----|----------|------|
| 新增協作者 | `POST /api/repos/:repo_id/collaborators` | `/repo/:id/collaborators` | ✅ |
| 列出協作者 | `GET /api/repos/:repo_id/collaborators` | `/repo/:id/collaborators` | ✅ |
| 移除協作者 | `DELETE /api/repos/:repo_id/collaborators/:user_id` | `/repo/:id/collaborators` | ✅ |

權限等級：`read` / `write` / `admin`。

### Secrets（加密）

CI/CD 環境變數，AES-256-GCM 加密儲存。

| 功能 | API | 前端頁面 | 狀態 |
|------|-----|----------|------|
| 新增 Secret | `POST /api/repos/:repo_id/secrets` | `/repo/:id/secrets` | ✅ |
| 列出 Secrets | `GET /api/repos/:repo_id/secrets` | `/repo/:id/secrets` | ✅ |
| 刪除 Secret | `DELETE /api/repos/:repo_id/secrets/:secret_id` | `/repo/:id/secrets` | ✅ |

加密金鑰來自 config.toml `[secrets] encryption_key`，若未設定則使用 `[jwt] secret` 作為 fallback。

### Branch Protection

分支保護規則（require PR、require approvals、dismiss stale reviews）。

| 功能 | API | 前端頁面 | 狀態 |
|------|-----|----------|------|
| 新增保護規則 | `POST /api/repos/:repo_id/branch-protections` | `/repo/:id/branch-protection` | ✅ |
| 列出保護規則 | `GET /api/repos/:repo_id/branch-protections` | `/repo/:id/branch-protection` | ✅ |
| 刪除保護規則 | `DELETE /api/repos/:repo_id/branch-protections/:protection_id` | `/repo/:id/branch-protection` | ✅ |

## 實作細節

### 資料表

```sql
CREATE TABLE access_tokens (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id     INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    name        TEXT NOT NULL,
    token_prefix TEXT NOT NULL DEFAULT '',
    token_hash  TEXT NOT NULL,
    scopes      TEXT NOT NULL DEFAULT 'repo',
    last_used_at DATETIME,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    expires_at  DATETIME
);

CREATE TABLE repo_collaborators (
    repo_id     INTEGER NOT NULL REFERENCES repositories(id) ON DELETE CASCADE,
    user_id     INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    permission  TEXT NOT NULL DEFAULT 'write',
    PRIMARY KEY (repo_id, user_id)
);

CREATE TABLE repo_secrets (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    repo_id         INTEGER NOT NULL REFERENCES repositories(id) ON DELETE CASCADE,
    name            TEXT NOT NULL,
    encrypted_value BLOB NOT NULL,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(repo_id, name)
);

CREATE TABLE branch_protection (
    id                      INTEGER PRIMARY KEY AUTOINCREMENT,
    repo_id                 INTEGER NOT NULL REFERENCES repositories(id) ON DELETE CASCADE,
    pattern                 TEXT NOT NULL,
    require_pr              INTEGER NOT NULL DEFAULT 1,
    require_approvals       INTEGER NOT NULL DEFAULT 1,
    dismiss_stale_reviews   INTEGER NOT NULL DEFAULT 1,
    UNIQUE(repo_id, pattern)
);
```

### 加密實作 (AES-256-GCM)

```rust
use aes_gcm::{Aes256Gcm, Key, Nonce, AeadInPlace, AeadCore, KeyInit};

fn encrypt_secret(plaintext: &[u8]) -> Result<Vec<u8>, AppError> {
    let key_bytes = auth::get_encryption_key();  // SHA-256 hash of config key
    let key = Key::<Aes256Gcm>::from_slice(&key_bytes);
    let cipher = Aes256Gcm::new(key);
    let nonce = Aes256Gcm::generate_nonce(&mut OsRng);
    let mut buf = plaintext.to_vec();
    cipher.encrypt_in_place(&nonce, &[], &mut buf)?;
    let mut result = nonce.to_vec();
    result.extend_from_slice(&buf);
    Ok(result)
}
```

### Token Hash (SHA-256)

```rust
use sha2::{Sha256, Digest};

let mut hasher = Sha256::new();
hasher.update(token.as_bytes());
let hash = hex::encode(hasher.finalize());
```

## 前端頁面

| 頁面 | 路徑 | 說明 |
|------|------|------|
| SettingsTokensPage | `/settings/tokens` | 建立/列出/刪除 Access Token，建立後顯示一次 raw token |
| RepoSettingsCollaboratorsPage | `/repo/:id/collaborators` | 依使用者名稱新增/移除協作者 |
| RepoSettingsSecretsPage | `/repo/:id/secrets` | 新增/刪除加密 Secret（值為 password input） |
| RepoSettingsBranchProtectionPage | `/repo/:id/branch-protection` | 新增/刪除分支保護規則 |

**導航整合**：
- `/settings` 頁面新增「Access Tokens」按鈕
- `/repo/:id/settings` 頁面新增「協作者」「Secrets」「Branch Protection」按鈕
