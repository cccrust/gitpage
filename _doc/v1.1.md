# v1.1 — Docker 開發容器（v1.2 起改用每人獨立容器，見 v1.2.md）

每個使用者一台 Docker 開發容器，SSH 登入即可編譯、執行專案。

## 架構

```
gitpage 宿主機
┌────────────────────────────┐
│  data/repos/{user}/{r}.git │ ← bare repos (Git push/pull)
│  data/staging/{user}/{r}/  │ ← file manager 暫存區
└────────┬───────────────────┘
         │ mount volume
         ▼
單一容器 gitpage (port 8080 + 22)
┌────────────────────────────┐
│  gitpage server            │
│  sshd (alice, bob, …)      │
│                            │
│  /home/alice/              │
│    blog/                   │ ← SSH login 時自動 clone
│    hello-rust/             │
│    …                       │
│  /home/bob/                │
│    portfolio/              │
│    …                       │
│                            │
│  工具: uv+Python, rustup+  │
│        Rust, Node.js, npm,│
│        git, opencode       │
└────────────────────────────┘
```

## 檔案

| 檔案 | 說明 |
|------|------|
| `Dockerfile.base` | 開發工具基底映像檔（uv+Python, rustup+Rust, Node.js, opencode） |
| `Dockerfile` | 多階段建置：frontend(Node) → backend(Rust) → runtime |
| `entrypoint.sh` | 容器 entrypoint：SSH host keys、建立使用者、寫入 repo sync 腳本、啟動 sshd + gitpage |
| `.dockerignore` | 排除 build artifacts、git history |
| `run_docker.sh` | 建置（自動判斷 base/app 是否需 rebuild）+ 啟動容器 |
| `test_docker.sh` | Docker 內完整整合測試 |

## 容器生命週期

1. `run_docker.sh` → build base image（首次，下載工具鏈）→ build app image（每次，秒級）→ run container
2. container 啟動 → entrypoint.sh 建立 SSH 使用者 → 啟動 sshd → 啟動 gitpage
3. 使用者 SSH 登入 → `.bashrc` 自動執行 `.gitpage-sync` → clone 該使用者的 bare repos 到 `~/`
4. 使用者可在容器內開發（`cargo run`、`node`、`uv run`…）

## SSH

- 多使用者共用一個容器（alice, bob），各自有密碼
- SSH port 預設 2222（避免與宿主機 sshd 衝突）
- 設定：`SSH_USERS="alice:alice123,bob:bob123"` 環境變數
- container 重啟後 SSH host keys 會變，宿主機需 `ssh-keygen -R '[localhost]:2222'`

## 共用基底映像檔

- `Dockerfile.base` 建出 `gitpage-dev-base:latest`，內含所有開發工具
- `Dockerfile` 的 runtime stage 直接 `FROM gitpage-dev-base`，只拷貝 gitpage binary + config + entrypoint，數秒完成
- 工具鏈更新時只需重建 base：`docker build -t gitpage-dev-base:latest -f Dockerfile.base .`

## 開發工具

| 工具 | 安裝方式 |
|------|----------|
| Python 3.12 | uv `uv python install 3.12`，binary copy 到 `/usr/local/bin/python` |
| Rust 1.96 | rustup 系統安裝（`RUSTUP_HOME=/usr/local/rustup`），實際 toolchain binary symlink 到 `/usr/local/bin/` |
| Node.js 18 | apt package |
| npm | apt package |
| opencode | 官方 install script |
| git | apt package |

## 注意

- `/root/` 權限為 700，symlink 到 `/root/` 內的檔案會 Permission denied。解法：使用 `cp` 或安裝到 `/usr/local/` 下
- Rust toolchain binary 使用 `$ORIGIN` rpath 找 shared library，不能用 `cp` 只能 `ln -sf` 到實際 binary
- 容器重啟後 repo 不會自動 reclone（有 `.git` 就跳過），除非刪除 `~/<repo>` 目錄
- 現有 `run.sh`、`test.sh`、`seed.sh` 無任何 Docker 相依，保持向後相容
