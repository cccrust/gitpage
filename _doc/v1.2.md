# v1.2 — 每人獨立 Docker 容器 + Container Exec 部署

每個使用者註冊時自動建立獨立 Docker 容器（`gitpage-{username}`），應用部署透過 `docker exec` 在容器內執行，徹底隔離。

## 架構

```
宿主機
┌──────────────────────────────────────────────┐
│  gitpage server (port 8080)                   │
│  ┌─────────────────────────────────────┐      │
│  │  mode = "docker"                     │      │
│  │  user register → DockerManager       │      │
│  │  .ensure_user_container()            │      │
│  │  git push → deploy_app()             │      │
│  │    ├→ checkout_source (host)         │      │
│  │    ├→ exec_build (container)         │      │
│  │    └→ exec_start_detached (container)│      │
│  │  proxy /app/{user}/{repo}/*          │      │
│  │    → container IP:port               │      │
│  │  restart → restore_apps_on_startup() │      │
│  │    ├→ exec_check_status (running?)   │      │
│  │    └→ exec_start_detached if not     │      │
│  └─────────────────────────────────────┘      │
│                                               │
│  data/                                        │
│  ├── repos/{u}/{r}.git    ← bare repos       │
│  ├── staging/{u}/{r}/     ← file manager     │
│  └── apps/{u}/{r}/source/ ← checked out src  │
│                                               │
│  Docker bridge                               │
│  ├── gitpage-alice  (172.17.0.2)             │
│  │   ├── cmd: sleep infinity (always up)     │
│  │   ├── volume: gitpage-home-alice → /home/ │
│  │   ├── bind: data/apps/alice → /workspace  │
│  │   └── SSH: host :22501 → container :22    │
│  ├── gitpage-bob   (172.17.0.3)              │
│  │   └── SSH: host :22502 → container :22    │
│  └── gitpage-test  (172.17.0.4)              │
│      └── SSH: host :22503 → container :22    │
└──────────────────────────────────────────────┘
```

## 容器生命週期

1. 使用者註冊 → `ensure_user_container(username)` 建立 + 啟動容器
2. 容器以 `sleep infinity` 持續運行（等待 exec 指令）
3. Git push 觸發自動部署：
   - host 端 `git checkout` 到 `data/apps/{user}/{repo}/source/`
   - container 內 `exec_build`：`sh -c "npm install"`（workdir: `/workspace/{repo}/source/`）
   - container 內 `exec_start_detached`：`sh -c "node server.js"`（detached, 背景運行）
   - port 值寫入 `apps_config.port` 供 restart 還原
4. 查詢 App 狀態：`exec_check_status` → `lsof -i :port`
5. 停止 App：`exec_stop_app` → `lsof -ti :port | xargs kill -9`
6. 重啟 server → `restore_apps_on_startup()`：
   - 讀取 DB 中所有 `enabled = 1` 的 app
   - Docker 模式：`exec_check_status` 檢查，若不在則重新 `exec_start_detached`
   - Process 模式：跳過（host subprocess 已消失）

## 容器規格

| 項目 | 說明 |
|------|------|
| Image | `gitpage-dev-base:latest`（含 Python, Rust, Node.js, git, opencode）|
| CMD | `sleep infinity`（保持運行）|
| Network | bridge（預設）|
| Volume | `gitpage-home-{user}:/home/{user}`（持久化 home）|
| Bind | `data/apps/{user}:/workspace`（建置工作區）|
| SSH port | 22/tcp → host `{ssh_port_range_start..=ssh_port_range_end}` 自動分配 |

## SSH 埠號分配

使用者註冊時從設定範圍內分配唯一 host port，對應容器內 22/tcp：

- 範圍：`[docker] ssh_port_range_start` ~ `ssh_port_range_end`（預設 22001-22999）
- 分配邏輯：`find_free_port()` 掃描已用埠 + port_allocations map
- 重啟還原：`DockerManager.connect()` 時 `list_running_containers()` 重建 port map
- 不同使用者：各自獨立 port（如 alice:22501, bob:22502）

```
ssh -p 22501 alice@host    # 進入 alice 容器
ssh -p 22502 bob@host      # 進入 bob 容器
```

## 兩種模式

| mode | build | start | stop | proxy target | restart 還原 |
|------|-------|-------|------|-------------|-------------|
| `process`（預設） | host subprocess | host spawn | `kill` + `lsof` | `127.0.0.1:port` | 不支持（process 已消失）|
| `docker` | `docker exec` | `docker exec --detach` | `exec_stop_app` (lsof) | `container_ip:port` | auto re-deploy |

## 關鍵檔案

| 檔案 | 說明 |
|------|------|
| `src/docker.rs` | `DockerManager`：connect, container CRUD, exec 方法, SSH port 分配 |
| `src/deploy.rs` | `run_build`/`start_app`/`stop_app`/`deploy_app` 接受 `Option<&DockerManager>` 雙模式 |
| `src/app.rs` | `fallback_handler` 依 mode 決定 proxy 目標 |
| `src/config.rs` | `RuntimeConfig` + `DockerConfig`（含 ssh_port_range）|
| `src/main.rs` | DockerManager 初始化、`--config` CLI、`restore_apps_on_startup()` |
| `src/handlers/auth.rs` | 註冊時 call `ensure_user_container` |
| `src/db/models.rs` | `AppsConfig` 新增 `port` 欄位、`EnabledAppWithOwner` 結構 |
| `Dockerfile.base` | 開發工具基底映像檔 |
| `run_docker.sh` | 共用容器模式，`gitpage-ssh-keys` volume 持久化 host keys |
| `test_docker_mode.sh` | Docker runtime mode 整合測試（含 SSH port + restart restore）|

## 組態

```toml
[runtime]
mode = "docker"       # "process" 為向後相容模式

[docker]
base_image = "gitpage-dev-base:latest"
network = "bridge"
memory_limit = "1g"
cpu_shares = 512
ssh_port_range_start = 22001
ssh_port_range_end = 22999
```

## 路徑約定

所有儲存路徑以 `storage.base_path`（預設 `"data"`）為根：

```
data/
├── repos/{u}/{r}.git       ← repo_path()
├── staging/{u}/{r}/        ← staging_path()
└── apps/{u}/{r}/           ← app_workspace_dir()
```

容器內 `/workspace/` 對應宿主機 `data/apps/{user}/`，為 bind mount。

## 測試

```bash
./test.sh                    # 非 Docker 模式（mode = process）
./test_docker_mode.sh        # Docker mode 測試（mode = docker）
./test_docker.sh             # Docker 封裝測試（共用容器）
```

`test_docker_mode.sh` 測試項目：
1. Docker connectivity
2. 使用者註冊 → container 建立
3. SSH port 發布（範圍內、不同使用者不同 port）
4. Named volume 存在
5. Apps bind mount
6. Repo 建立 + git push
7. Container exec build + start
8. Container IP 取得
9. API 文件樹
10. **Server 重啟 → app 自動還原**（status: running + proxy 正常）
11. 第二使用者獨立容器 + 不同 SSH port

## 共用容器模式（v1.1 向後相容）

`run_docker.sh` / `test_docker.sh` 仍使用單一容器 + sshd + entrypoint.sh，支援：
- 多使用者 SSH 登入（alice, bob）
- **SSH host keys 持久化**：`gitpage-ssh-keys` named volume mount 到 `/etc/ssh/`，容器重建後 key 不變
