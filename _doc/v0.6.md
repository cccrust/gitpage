# v0.6 — App Hosting（類似 Vercel）

讓 gitpage 能從 repository 部署 server-side 應用程式，支援 Node.js 與 Rust，透過 reverse proxy 提供 `/app/{username}/{repo}/` 存取。

---

## 架構概覽

```
git request → git push
       ↓
  auto_deploy_app()    ← 背景觸發部署
       ↓
  checkout → build → start process
       ↓
  /app/{user}/{repo}/* → proxy → http://127.0.0.1:{port}/{subpath}
```

每個 app 配置一個獨立 port（4000–65535），由 `AppProcessManager` 管理生命週期。

## 1. 資料庫：`apps_config` 表

```sql
CREATE TABLE IF NOT EXISTS apps_config (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    repo_id         INTEGER NOT NULL UNIQUE REFERENCES repositories(id),
    branch          TEXT DEFAULT 'main',
    source_dir      TEXT DEFAULT '/',
    build_command   TEXT DEFAULT '',
    start_command   TEXT DEFAULT '',
    env_vars        TEXT DEFAULT '{}',
    enabled         INTEGER DEFAULT 0
);
```

DB 操作（`src/db/mod.rs`）：
- `upsert_apps_config(repo_id, branch, source_dir, build_command, start_command, env_vars, enabled)`
- `get_apps_config(repo_id) → Option<AppsConfig>`
- `delete_apps_config(repo_id)`
- `get_enabled_apps() → Vec<AppsConfig>`

## 2. 專案類型自動偵測

實作於 `src/deploy.rs` 的 `resolve_commands()`。

| 類型 | 偵測條件 | 預設 build | 預設 start |
|------|----------|-------------|-------------|
| Node.js | `package.json` 存在 | `npm install` | `npm start` |
| Rust | `Cargo.toml` 存在 | `cargo build --release` | `./target/release/{binary}` |

Node.js 從 `package.json` 讀取 `scripts.start` 作為預設 start command。
Rust 從 `Cargo.toml` 的 `[package] name = "..."` 解析 binary 名稱（split on `=`）。

## 3. 建置與部署流程

實作於 `src/deploy.rs` 的 `deploy_app()`。

```
1. 建立 workspace: data/apps/{username}/{repo}/
2. 從 bare repo checkout 指定 branch 到 source/
   git --work-tree={workspace}/source --git-dir={bare_repo_path} checkout -f {branch}
3. 偵測專案類型（package.json / Cargo.toml）
4. 執行 build_command（若無則跳過）
5. 分配 port
6. 以 subprocess 啟動 start_command
   - 設定環境變數 PORT={port} 及其他自訂 env_vars
7. 註冊到 AppProcessManager
8. 啟動監控 task，process 退出時更新狀態為 Stopped/Error
```

### Node.js 範例
```bash
# checkout → source/
npm install            # build
PORT=4000 npm start    # start (server.js 監聽 PORT env)
```

### Rust 範例
```bash
# checkout → source/
cargo build --release                # build
PORT=4000 ./target/release/myapp    # start
```

## 4. Process Manager

定義於 `src/deploy.rs`。

```rust
pub struct AppProcessManager {
    processes: Arc<Mutex<HashMap<i64, AppProcess>>>,
    next_port: Arc<AtomicU16>,
}

pub struct AppProcess {
    repo_id: i64,
    port: u16,
    child: Option<tokio::process::Child>,
    status: AppStatus,
}

pub enum AppStatus {
    Deploying,
    Running,
    Stopped,
    Error(String),
}
```

### 功能
- `allocate_port() → u16` — 從 4000 開始找第一個可用 port
- `start(repo_id, username, repo_name, config) → Result` — 建置 + 啟動
- `stop(repo_id) → Result` — 終止 process
- `get(repo_id) → Option<AppProcess>`
- `list() → Vec<AppProcess>`

### 已知限制
- **Process 狀態為 in-memory**：server 重啟後所有 app 狀態顯示 stopped，需手動 redeploy
- Process crash 後不會自動重啟（僅更新狀態）

## 5. Reverse Proxy

實作於 `src/app.rs` 的 `fallback_handler`。

```
/app/{username}/{repo_name}/{*path}
```

步驟：
1. 從 path 解析 username 與 repo_name
2. 查 DB 取得 user_id 與 repo_id
3. 從 `AppProcessManager` 取得 port
4. 用 `reqwest::Client` 代理請求到 `http://127.0.0.1:{port}/{subpath}`
5. **正確轉發 HTTP method 與 body**（PUT/POST/DELETE 不再被轉為 GET）
6. 回傳 upstream 的 status code、Content-Type 與 body

```rust
let reqwest_req = client
    .request(reqwest::Method::from_bytes(method.as_str().as_bytes()).unwrap_or(reqwest::Method::GET), &url)
    .body(body_bytes)
    .build()?;
```

## 6. API

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | /api/apps/:repo_id | 取得 app config + 執行狀態 |
| PUT | /api/apps/:repo_id | 新增/更新 app config，啟用時自動部署 |
| POST | /api/apps/:repo_id/deploy | 手動觸發部署（build + restart）|
| DELETE | /api/apps/:repo_id | 停止並移除 app config |

### GET /api/apps/:repo_id response
```json
{
  "apps_config": { "id": 9, "repo_id": 8, "branch": "main", ... },
  "status": "running",
  "port": 4002,
  "url": "/app/alice/todo-node"
}
```

### PUT /api/apps/:repo_id request
```json
{
  "branch": "main",
  "source_dir": "/",
  "build_command": "npm install",
  "start_command": "",
  "env_vars": "{}",
  "enabled": true
}
```

## 7. 自動部署

在 `app.rs` 的 `fallback_handler` 中，git push 成功後 spawn `auto_deploy_app`：

```rust
if is_push && resp.status().is_success() {
    tokio::spawn(auto_deploy_pages(state.clone(), username.to_string(), repo_name.to_string()));
    tokio::spawn(auto_deploy_app(state.clone(), username.to_string(), repo_name.to_string()));
}
```

`auto_deploy_app()` 邏輯：
1. 查 DB 找 user/repo
2. 查 apps_config，若 enabled=true
3. 呼叫 `deploy_app()` 執行完整部署流程

## 13. 已知問題

- **Process 狀態無持久化**：server 重啟後需手動 redeploy（DB config 保留，但 in-memory 狀態遺失）
- **無快取層**：每次 deploy 都重新 `npm install` / `cargo build`
- **無日誌串流**：目前無法從 UI 查看 app 執行日誌
- **無容器隔離**：所有 app 跑在同一 process namespace
- **Rust 構建較慢**：每次 deploy 需完整 `cargo build --release`
