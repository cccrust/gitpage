# v0.9 — SSH Shell（透過 SSH 連線進專案目錄）

讓使用者可以透過 SSH 連線到伺服器，自動進入 repo 的 staging 目錄，直接在 terminal 中操作（npm install、cargo build、git 指令等），就像在本地開發一樣。

---

## 架構概覽

```
使用者終端機
  │
  │  ssh git@host
  │  (或 ssh git@host "npm install")
  │
  ▼
系統 OpenSSH Server (port 22 或自訂)
  │
  │  AuthorizedKeysFile → data/ssh/authorized_keys
  │  每筆 entry 有 command="/path/to/handler.sh user repo" 前綴
  │
  ▼
gitpage-shell handler script
  │
  │  cd /data/staging/{user}/{repo}/
  │  exec bash -i  (或 bash -c "$SSH_ORIGINAL_COMMAND")
  │
  ▼
使用者在專案目錄中操作
```

### 連線方式

使用者將 SSH public key 上傳到平台（每個 key 綁定一個 repo），之後：

```bash
# 互動式 shell（自動進入專案目錄）
ssh git@host

# 單一指令執行（等同在該目錄下執行）
ssh git@host "npm run build"

# 強制 PTY（需要互動式的單一指令）
ssh -t git@host "npm run dev"
```

## 1. Database

### ssh_keys 表

```sql
CREATE TABLE IF NOT EXISTS ssh_keys (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id    INTEGER NOT NULL REFERENCES users(id),
    repo_id    INTEGER NOT NULL REFERENCES repositories(id),
    name       TEXT NOT NULL DEFAULT '',
    public_key TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

| 欄位 | 說明 |
|------|------|
| `id` | 自動遞增主鍵 |
| `user_id` | 外鍵，指向使用者 |
| `repo_id` | 外鍵，指向 repository（一個 key 只綁定一個 repo）|
| `name` | 使用者自訂名稱（如 "我的 MacBook"）|
| `public_key` | SSH public key 全文（`ssh-rsa AAAA...` 或 `ssh-ed25519 AAAA...`）|

## 2. SSH authorized_keys 管理

### 產生 authorized_keys

每次 ssh_keys 表有變動時，應用程式重新產生 `data/ssh/authorized_keys`：

```
command="/Users/cccuser/.ssh/gitpage-shell" "alice" "hello-node",no-port-forwarding,no-X11-forwarding,no-agent-forwarding ssh-ed25519 AAAAC3... alice@macbook
command="/Users/cccuser/.ssh/gitpage-shell" "alice" "hello-rust",no-port-forwarding,no-X11-forwarding,no-agent-forwarding ssh-rsa AAAAB3... bob@workstation
command="/Users/cccuser/.ssh/gitpage-shell" "bob" "blog",no-port-forwarding,no-X11-forwarding,no-agent-forwarding ssh-ed25519 AAAAC4... bob@thinkpad
```

限制選項：
- `no-port-forwarding` / `no-X11-forwarding` / `no-agent-forwarding`：禁止進階 SSH 功能
- `no-pty`：預設不分配 PTY（但 `ssh -t` 仍可強制分配）

### gitpage-shell handler script（啟動時自動寫入 `data/ssh/gitpage-shell`）

安裝於 `~/.ssh/gitpage-shell`（由應用程式在啟動時自動產生並設為可執行）：

```bash
#!/bin/bash
# 參數：$1 = username, $2 = repo_name
USERNAME="$1"
REPO_NAME="$2"
STAGING_DIR="/absolute/path/to/gitpage/data/staging/$USERNAME/$REPO_NAME"

if [ ! -d "$STAGING_DIR" ]; then
    echo "ERROR: Staging directory not found: $STAGING_DIR"
    exit 1
fi

cd "$STAGING_DIR" || exit 1

if [ -n "$SSH_ORIGINAL_COMMAND" ]; then
    # 非互動模式：執行使用者的指令
    exec bash -c "$SSH_ORIGINAL_COMMAND"
else
    # 互動模式：啟動 interactive bash
    exec bash -i
fi
```

### 系統設定

應用程式會自動寫入 `~/.ssh/authorized_keys` 與 `~/.ssh/gitpage-shell`（啟動時及每次增刪 SSH key 時）。

只需確認系統的 SSH Remote Login 已開啟：

```bash
# macOS
sudo systemsetup -setremotelogin on

# Linux
sudo systemctl enable --now sshd
```

## 3. API

### SSH Key CRUD

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | /api/repos/:id/ssh-keys | 列出該 repo 的 SSH keys |
| POST | /api/repos/:id/ssh-keys | 新增 SSH key（body: `{ name, public_key }`）|
| DELETE | /api/repos/:id/ssh-keys/:key_id | 刪除 SSH key |

### POST /api/repos/:id/ssh-keys request

```json
{
    "name": "我的 MacBook Pro",
    "public_key": "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAINw..."
}
```

### POST /api/repos/:id/ssh-keys response

```json
{
    "success": true,
    "ssh_key": {
        "id": 1,
        "name": "我的 MacBook Pro",
        "public_key": "ssh-ed25519 AAAAC3...",
        "created_at": "2026-06-16T12:00:00Z"
    }
}
```

### 內部：authorized_keys 重新產生

每次增刪 SSH key 後，`SshKeyManager` 重新掃描 DB 中所有 keys，重新寫入 `data/ssh/authorized_keys`。

## 4. 後端實作

### Cargo.toml 異動

無需新增外部 crate。使用既有 `rusqlite` 與 `std::fs` 即可。

### 新檔案：`src/handlers/ssh_keys.rs`

```rust
pub async fn list_keys(...)
pub async fn add_key(...)
pub async fn delete_key(...)
```

### 新模組：`src/ssh.rs`

負責 authorized_keys 的產生：

```rust
pub async fn regenerate_authorized_keys(db: &Database, output_path: &str) -> Result<()> {
    let all_keys = db.get_all_ssh_keys().await?;
    let mut output = String::new();
    for key in all_keys {
        let user = db.find_user_by_id(key.user_id).await?;
        let repo = db.get_repo(key.repo_id).await?;
        let command = format!(
            "command=\"/usr/local/bin/gitpage-shell {} {}\"",
            user.username, repo.name
        );
        let restrictions = "no-port-forwarding,no-X11-forwarding,no-agent-forwarding";
        writeln!(output, "{},{},{} {}", command, restrictions, key.public_key, key.name)?;
    }
    std::fs::write(output_path, output)?;
    Ok(())
}
```

## 6. 安全性

- 僅 repo owner 可管理該 repo 的 SSH keys
- SSH public key 格式驗證（必須以 `ssh-rsa`、`ssh-ed25519`、`ecdsa-sha2-*` 等開頭）
- authorized_keys 的 `command=` 限制無法繞過（OpenSSH 強制執行）
- `no-pty` 等 restriction 防止進階 SSH 功能
- 刪除 key 後立即重新產生 authorized_keys，舊 key 即時失效

- OpenSSH Server（系統已安裝）

## 9. 注意事項

- 應用程式自動管理 `~/.ssh/authorized_keys`，無需手動設定
- 需確認 SSH Remote Login 已啟用（`sudo systemsetup -setremotelogin on`）
- PTY 支援依賴 `ssh -t` 參數；無 `-t` 時 bash 以 non-interactive 模式執行（無顏色提示、無 tab 補全）
